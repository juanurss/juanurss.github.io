The last 27,28 and 29 of September was the ekoparty security confence,in this conference I did a workshop focused on mobile application implementations. And I talked about some security examples and implementations where thoose examples were based on differents mobile penetration test what I did.

Where I showed some "bad implementations" inside the code, hardcoded issues like usernames, passwords, services (FTP, SFTP, etc) or the lack of binary protections like obfuscation, anti-emulation, root checker, bad implementations of certificate pinning, SSL/TLS, etc.
